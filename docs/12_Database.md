# 12 — Database

The Engine's local state store: **SQLite**, embedded, file-based, transactional, zero-ops. It holds
**coordination** state (locks, runs) and the **Brain** ([04](04_ProjectBrain.md)). It is **not** an
authority on what work exists (Jira is, P2) or on code (Git is, P3) — it caches those and stores
execution/knowledge state, and is **rebuildable** (NFR-7).

## Why SQLite

- Embedded, single file, no server → local-first (P1) and simple (P6, NFR-4).
- ACID transactions → safe locking and crash recovery (NFR-8).
- Trivially lives on the SSD (C-6). One DB file per project.

## Location

```
<engine-home>/projects/<project>/brain.db      # per project (coordination + brain)
<engine-home>/engine.db                         # engine-global (projects registry, global settings)
```

`<engine-home>` defaults to `/Volumes/Extreme SSD/cwv2-home/` (C-6, configurable).

### Brain — index & graph (rebuildable, [04](04_ProjectBrain.md))

**`files`** (path, repo, lang, size, hash, summary, indexed_at) —
**`symbols`** (id, file_id, kind, name, signature, line) —
**`deps`** (from_file_id, to_file_id, kind) — deterministic import/reference edges.

### Brain — knowledge (authored; the irreplaceable part)

**`decisions`** — mirror of `knowledge/decisions/NNNN-*.md` for querying (id, title, context,
decision, consequences, issue_key, created_at). Markdown file is the human-facing copy; row is the
index.

**`failures`** — known failures (id, signature, detail, resolution, first_seen, last_seen, count,
flaky bool). Feeds "current failures" prompt slice.

**`deferrals`** — (id, issue_key, kind, reason, howto, env_needed, followup_key, created_at,
resolved_at). Feeds dashboard + sweeps ([03](03_Workflow.md)/[06](06_QA.md)/[10](10_Hardware.md)).

**`qa_maps`** — screens/nav (id, screen, reach_path, golden_artifact, updated_at). Golden images are
files on disk; row references them.

### Engine-global (`engine.db`)

**`projects`** — registry (name, config_path, repos, engine_home_subdir, enabled).
**`settings`** — global knobs (usage thresholds, default concurrency) overridable per project config.

## Transactions & locking semantics

- Lock acquire = a single transaction: insert-if-absent-or-stale. Heartbeats update `heartbeat_at`.
- The **merge lock** is a single well-known row (`merge:<repo>`); the Integrator holds it for the
  duration of a merge, guaranteeing serialized advancement of `development` (FR-8, I-3).
- Reaper job (scheduled, deterministic) deletes locks whose `heartbeat_at + ttl` < now and triggers
  cleanup of the associated worktree/run (NFR-8).

## Rebuildability (NFR-7)

- **Index/graph/qa-maps** → regenerated by deterministic indexers from the repo.
- **issues_cache** → refetched from Jira.
- **runs/events** → historical; loss only loses history, not correctness (Jira + Git hold truth).
- **decisions/failures/deferrals** → the authored knowledge; backed by markdown (`decisions/`) and
  by Jira follow-up issues (deferrals). A periodic export writes these to the repo/knowledge dir so
  they survive DB loss (aligns with the owner's memory-backup practice).

## Migrations & integrity

- Schema versioned; forward-only migrations run at startup.
- `PRAGMA foreign_keys=ON`, WAL mode for concurrent readers (dashboard) + one writer (engine).
- Nightly `VACUUM`/integrity check; corruption → rebuild from repo + Jira + knowledge markdown.

## What the DB never stores

- Secrets (keychain/Azure KV only, NFR-6).
- The authoritative list of work (Jira).
- Code (Git).
- Anything that can't be rebuilt *and* isn't also persisted to Jira/Git/knowledge-markdown.
